## 9. 스프링 트랜잭션 이해

### 1. 트랜잭션 적용 위치
- 스프링 우선순위 : 더 구체적이고 자세한 것이 높은 우선순위를 가짐
  - 우선순위
  - 클래스에 적용 시 메서드는 자동 적용
- 인터페이스 @Transactional 적용
  - 클래스의 메서드
  - 클래스의 타입
  - 인터페이스의 메서드
  - 인터페이스의 타입

### 2. 트랜잭션 AOP 주의 사항 - 프록시 내부 호출 1
- 대상 객체의 내부에서 메서드 호출 발생 -> 프록시를 거치지 않고 대상 객체 직접 호출
- public 메서드만 트랜잭션 적용
